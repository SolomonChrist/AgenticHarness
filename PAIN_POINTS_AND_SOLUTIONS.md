# Pain Points From Traditional Harnesses

Simple outline for promotional slides, PDF, or HTML.

## Core Pain Points

### 1. Vendor lock

Pain:

- the work only really lives inside one vendor's runtime
- switching providers feels risky
- token limits and pricing changes can break momentum

Agentic Harness answer:

- the system lives in local files
- any compatible harness can take over
- roles, tasks, and memory survive vendor swaps

### 2. Fear of losing work

Pain:

- users do not know what to back up
- important context is trapped in chats or hidden runtime state
- when bots stop, work feels inaccessible

Agentic Harness answer:

- work is stored in inspectable folders
- memory is markdown
- tasks are markdown
- recovery is file-based, not chat-history-based

### 3. Hard to restart after failure

Pain:

- bots die
- sessions expire
- the machine restarts
- nobody knows how to recover the system

Agentic Harness answer:

- leases show what roles were active
- event logs show what happened last
- memory shows what each role knew
- a new harness can take over from files alone

### 4. Too much hidden complexity

Pain:

- systems become “working black boxes”
- the user does not understand the folder structure or moving parts
- growth creates anxiety instead of confidence

Agentic Harness answer:

- the control plane is explicit
- the layers are named
- the system map is understandable
- the user can inspect everything directly

### 5. Overwhelm when scaling

Pain:

- more work arrives than one person can manage
- users do not know which bots to create next
- it becomes unclear who should do what

Agentic Harness answer:

- `Chief_of_Staff` recommends roles
- roles are explicit and replaceable
- Runner keeps the swarm alive
- memory compounds over time

### 6. Poor visibility

Pain:

- there is no easy way to see what the swarm is doing
- status lives across scattered sessions
- it is hard to tell whether the system is healthy

Agentic Harness answer:

- file-first status is always available
- Telegram gives remote access
- Visualizer adds 2D / 3D / VR visibility on top of the same files

### 7. Fear of starting over on a new machine

Pain:

- if the computer dies, the user fears they must rebuild everything from scratch
- valuable preferences and workflow knowledge disappear

Agentic Harness answer:

- back up the core, memory, projects, and runner
- restore the folder
- boot `Chief_of_Staff`
- continue from the files

## Simple Messaging

Traditional harnesses often make users feel:

- locked in
- dependent
- overwhelmed
- one crash away from chaos

Agentic Harness aims to make them feel:

- portable
- backed up
- restartable
- understandable
- in control

## One-Line Positioning

Agentic Harness is the backup-friendly, vendor-flexible control plane for AI work that has to survive real life.
