# RECOVERY CHECKLIST

Use this when:

- bots died
- sessions were lost
- the machine restarted
- you changed vendors
- you are restoring on a new machine

## 1. Confirm The Files Exist

Check for:

- `AGENTIC_HARNESS.md`
- `LAYER_TASK_LIST.md`
- `LAYER_MEMORY.md`
- `LAYER_LAST_ITEMS_DONE.md`
- `ROLES.md`
- `HUMANS.md`
- `MEMORY/`
- `Projects/`

## 2. Confirm Optional Layers

If you use them, also check:

- `Runner/`
- `TelegramBot/`
- `Visualizer/`

## 3. Boot One Trusted Harness

Paste:

```text
Read AGENTIC_HARNESS.md first.
Take the role of Chief_of_Staff if it is open or stale.
```

If needed, use the tiny path instead:

```text
Read AGENTIC_HARNESS_TINY.md first.
Take the role of Chief_of_Staff if it is open or stale.
```

## 4. Have Chief_of_Staff Reconstruct State

It should read:

- `MEMORY/agents/Chief_of_Staff/ONBOARDING_STATUS.md`
- operator memory
- `LAYER_TASK_LIST.md`
- `LAYER_MEMORY.md`
- `LAYER_LAST_ITEMS_DONE.md`
- recent project files

## 5. Re-establish Liveness

If used:

- start Telegram bridge
- start Runner

## 6. Reclaim Specialist Roles As Needed

Bring roles back one at a time:

- Researcher
- Engineer
- Documentation
- etc.

## 7. Do Not Panic About Old Sessions

Old sessions are disposable.

What matters is:

- the files
- the memory
- the task board
- the recovery path

That is the whole point of Agentic Harness.
